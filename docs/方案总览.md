# 方案总览

## 目标

本仓库用于统一管理多套信号处理实现方案，避免把所有代码直接堆在一个大仓库中。

## 当前方案

### H7

- 方案定位：`STM32H7` 主控方案
- 当前状态：已接入总控仓库，当前仍使用本地仓库路径
- 子模块路径：`H7/`
- 后续动作：推送到 GitHub 后切换为正式远程子模块 URL

### F4

- 方案定位：`STM32F4` 方案
- 当前状态：预留
- 目标路径：`F4/`

### TI

- 方案定位：`TI` 平台方案
- 当前状态：预留
- 目标路径：`TI/`

### F4_FPGA

- 方案定位：`STM32F4 + FPGA` 协同方案
- 当前状态：已接入总控仓库，当前仍使用本地仓库路径
- 子模块路径：`F4_FPGA/`
- 旧代码归档：`F4_FPGA/old_code.zip`
- 后续动作：推送到 GitHub 后切换为正式远程子模块 URL

## 选择这种结构的原因

- 各方案可以独立开发、独立版本化
- 总控仓库体量更可控
- 便于统一管理跨方案文档、规范和版本关系
- 后续每套方案都可以按自己的技术栈演进
