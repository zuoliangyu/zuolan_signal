# 子模块协作规范

## 1. 根仓库与子模块的职责

### 根仓库负责

- 子模块接入与路径管理
- 方案总览与导航
- 跨方案规则
- 版本映射与集成记录

### 子模块负责

- 具体方案代码
- 本方案 `AGENTS.md`
- 本方案文档、构建说明、验证说明
- 本方案独立提交与版本演进

## 2. 新增子模块的基本流程

1. 先准备独立仓库
2. 在根仓库执行 `git submodule add <repo-url> <path>`
3. 提交 `.gitmodules` 和子模块目录指针
4. 更新 `README.md`、`docs/README.md`、`docs/方案总览.md`

## 3. 更新已有子模块的流程

1. 进入子模块目录完成开发与提交
2. 推送子模块仓库
3. 回到根仓库提交新的子模块指针
4. 如有路径、接口、流程、脚本变化，同步更新相关文档

## 5. AGENTS 规则分层

### 根目录 `AGENTS.md`

- 只处理总控层规则
- 不写死具体芯片方案的构建或烧录命令

### 子模块 `AGENTS.md`

- 处理本方案技术栈、目录、验证、文档要求
- 保证单独打开子模块仓库时也能独立工作

## 6. 禁止事项

- 不要把子模块目录加入根仓库 `.gitignore`
- 不要只在根仓库写关键方案规则而不在子模块补齐
- 不要在根仓库长期堆放应属于子模块的大量源码
