# 总控仓库协作约定

## 仓库定位

- 本仓库是 `zuolan_signal` 总控仓库。
- 默认只承载方案导航、跨方案文档、版本映射、公共脚本与协作规则。
- 各具体方案代码仓库以子模块形式接入，原则上不把大体量业务代码长期直接堆在根仓库。

## 默认语言

- 默认使用简体中文。
- 仅在命令名、API 名、文件名、专有名词等必须保留英文时使用英文。

## 子模块协作边界

- 根目录规则只约束总控仓库和跨方案事项。
- 每个子模块必须维护自己的 `AGENTS.md`，用于约束本方案内部的开发、验证、文档和目录规范。
- 不要把仅对子模块有效的构建命令、烧录方式、IDE 约束写死在本仓库根规则中。

## 当前方案布局

- `H7/`：`STM32H7` 方案子模块
- `F4/`：预留
- `TI/`：预留
- `F4_FPGA/`：预留

## 文档与版本要求

- 方案增删、路径调整、子模块 URL 变更时，必须同步更新：
  - `README.md`
  - `docs/README.md`
  - `docs/子模块协作规范.md`
  - `docs/方案总览.md`
- 根仓库提交子模块指针变更时，提交说明需明确：
  - 哪个子模块变了
  - 变更目的
  - 是否需要同步拉取或初始化子模块

## Git 约定

- 不要把子模块目录加入根仓库 `.gitignore`。
- 新增子模块时，必须提交 `.gitmodules`。
- 更新子模块后，回到根仓库同步提交子模块指针。
- 如果子模块当前仍是本地路径接入，文档中必须明确后续切换到正式远程地址的步骤。

## 验证约定

- 默认不自动执行全量构建、全量测试、烧录或重型验证。
- 本仓库以结构、文档、Git 状态校验为主。

## 当前阶段特别说明

- 当前 `H7` 子模块先按本地仓库接入。
- 待 `H7` 独立仓库推送到 GitHub 后，再把 `.gitmodules` 中的本地 URL 切换为正式远程 URL。
